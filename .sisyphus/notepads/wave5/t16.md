## T16: Error Handling + Edge Cases - Implementation Summary

### Date: 2026-02-27

### Implemented Features

#### 1. Global Hotkey Conflict Detection
- **File**: `src-tauri/src/window.rs` (register_summon_hotkey)
- **Behavior**: If hotkey registration fails (e.g., Cmd+Shift+Space in use by another app), sends macOS notification: "Weave: Hotkey Registration Failed - Cmd+Shift+Space is in use by another app. Please change the hotkey in settings."
- **Implementation**: `.map_err()` on `register()` call triggers notification via new `notifications` module

#### 2. Accessibility Permissions Notification
- **Files**: 
  - `src-tauri/src/notifications.rs` (NEW module)
  - `src-tauri/src/window.rs` (position_window_near_cursor)
- **Behavior**: On first cursor position failure (accessibility denied), shows one-time notification: "Weave: Accessibility Permission Required - Go to System Settings > Privacy & Security > Accessibility and grant Weave access for cursor positioning."
- **Storage**: Uses `tauri-plugin-store` with key `notification_accessibility_shown` to prevent repeat notifications
- **Fallback**: Window positioning fails gracefully (returns early), doesn't crash app

#### 3. Auto-Paste Reliability
- **File**: `src-tauri/src/clipboard.rs` (auto_paste_macos)
- **Changes**:
  - Increased delay from 150ms to 200ms for app switching stability
  - Added 150ms additional delay after app activation for paste command
  - Wrapped osascript execution in error handling with eprintln fallback message
- **Fallback**: If paste command fails, content remains in clipboard for manual paste (silent degradation)
- **Edge case**: Empty content now skips clipboard write and just dismisses window

#### 4. Window Pool Edge Cases
- **File**: `src-tauri/src/window.rs`
- **Changes**:
  - **Pool initialization failure**: Changed `if let Ok()` to `match` with error logging. App continues with on-demand window creation if pre-creation fails
  - **Max window cap**: Added `MAX_WINDOWS = 10` constant. `show_pooled_window` now counts active editor windows and returns error if >= 10: "Maximum 10 windows reached. Close some windows first."
  - **Rapid hotkey presses**: Cap prevents resource exhaustion from spam-pressing hotkey
- **Tests**: Added tests for max cap, saturating_sub overflow prevention, and initialization failure handling

#### 5. AI Error Handling
- **File**: `src-tauri/src/ai.rs` (format_text)
- **Changes**:
  - **Network timeout**: Added 30s timeout to reqwest client. Timeout error returns: "Network timeout. Check your connection and try again."
  - **Connection error**: Returns: "Network error. Check your internet connection."
  - **HTTP 401**: Returns: "Invalid API key. Check your OpenAI API key in settings."
  - **HTTP 429**: Returns: "API rate limit reached. Try again in a moment."
  - **HTTP 5xx**: Returns: "OpenAI service error. Try again later."
  - **Invalid JSON response**: Returns: "Unexpected response from AI. Try again. (error details)"
  - **Empty content**: Returns immediately with default response (no API call)
- **Content preservation**: All error paths ensure original editor content is never lost

#### 6. Empty Editor Edge Cases
- **File**: `src-tauri/src/clipboard.rs` (copy_and_dismiss)
- **Behavior**: If content is empty, skips clipboard write and just dismisses window (no clipboard change)
- **File**: `src-tauri/src/ai.rs` (format_text)
- **Behavior**: If content is empty or whitespace-only, returns immediately without API call

#### 7. New Module: notifications.rs
- **Purpose**: Centralized notification management with one-time notification tracking
- **Functions**:
  - `send_notification()`: macOS notification via `tauri-plugin-notification`
  - `has_shown_notification()`: Check if notification key exists in store
  - `mark_notification_shown()`: Persist notification flag to store
  - `send_accessibility_notification_once()`: Convenience wrapper for accessibility prompt
- **Tests**: 4 tests covering key format, title/body content, and naming conventions

### Technical Notes

#### Pattern: try_state Returns Option, Not Result
- `window.app_handle().try_state::<T>()` returns `Option<State<T>>`, not `Result`
- Must use `if let (Some(app), Some(store)) = ...` not `if let (Ok(app), Ok(store)) = ...`

#### Pattern: Notification Store Keys
- Convention: `notification_{feature}_shown` for one-time notifications
- Stored as `serde_json::json!(true)` in `tauri-plugin-store`

#### Pattern: Silent Degradation for Non-Critical Features
- Auto-paste failure: eprintln + continue (content in clipboard for manual paste)
- Cursor position failure: return early + notification (fallback to center positioning)
- Pool creation failure: log error + continue (on-demand window creation)

#### Pattern: Error Message User-Friendliness
- Network errors: "Check your connection" (not raw reqwest error)
- Auth errors: "Check your API key in settings" (actionable)
- Rate limits: "Try again in a moment" (temporary issue)
- Generic: Include technical details only in parentheses

### Edge Cases Covered

✅ Accessibility permission denied
✅ Auto-paste command failure
✅ Window pool creation failure on startup

### Known Limitations
- Hotkey conflict notification triggers once per app launch (doesn't retry)
- Accessibility notification is one-time only (user must manually re-enable if dismissed)
- Max window cap is hardcoded at 10 (not user-configurable)
- Auto-paste delay is fixed at 200ms + 150ms (not adaptive)
